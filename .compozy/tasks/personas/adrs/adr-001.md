# ADR-001: Dedicated Personas Page MVP

## Status

Accepted

## Date

2026-07-07

## Context

Users need a clearer way to manage personas used by Promptizer generation. The app already supports persona selection and custom persona create/edit/delete flows, but management is distributed through modal interactions. The requested product outcome is more consistent generated prompts by making persona descriptions easier to view, create, edit, delete, and intentionally select before generation.

## Decision

Create a simple dedicated Personas page inside Promptizer for the MVP. The page will let users view existing personas, create new personas, edit existing personas, and delete personas. The app will start with two personas, and both can be edited or deleted. Each persona will have only a title and description. The selected persona description will be used as the behavior context when the user clicks Generate. If no persona exists or no usable persona can be selected, Generate will be blocked with clear guidance to create or select a persona.

## Alternatives Considered

### Alternative 1: Contextual Management in Current Screen

- **Description**: Keep persona management inside the current Promptizer screen and improve inline actions or modal flows.
- **Pros**: Smaller navigation change; keeps all actions close to generation.
- **Cons**: Less room for comparing and managing personas; can keep the existing workflow feeling scattered.
- **Why rejected**: The user explicitly wants a page for seeing, creating, and editing personas.

### Alternative 2: Personas Page with Testing Preview

- **Description**: Add full CRUD plus a preview or test area to try a persona before using it in Generate.
- **Pros**: Helps users validate persona quality before generation; reduces prompt inconsistency.
- **Cons**: Larger MVP scope; introduces extra interaction design before the core management flow is proven.
- **Why rejected**: The MVP should stay focused on title, description, CRUD, and generation behavior.

## Consequences

### Positive

- Users get one clear place to manage personas.
- The feature directly supports the desired outcome of more consistent prompt generation.
- MVP scope stays small and testable.
- Editing and deleting the two initial personas keeps the model simple: every persona is user-manageable.

### Negative

- Without a test preview, users must evaluate persona quality through the normal Generate flow.
- Blocking Generate when no persona exists adds friction after users delete all personas.

### Risks

- Users may not understand that the description controls generation behavior. Mitigate with clear helper text on the page.
- Users may delete all personas and feel stuck. Mitigate with an empty state that prompts creation before generation.
- Short descriptions may be too vague for reliable prompt behavior. Mitigate with concise guidance in the description field.

## Implementation Notes

Implementation details belong in the TechSpec. Product-level constraints: the page belongs inside Promptizer, personas have only title and description for MVP, and Generate requires an available selected persona.
